## Your main question: “If admin creates form, those forms will be stored right? Every form is different… how?”

Yes — forms are stored, but **not as new database tables each time**.

### The right industry approach (simple)

You store a “form definition” as data, like this:

#### 1) FormTemplate (the blueprint)

Example: “Daily Safety Checklist”

* id
* title
* plantId (optional)
* version
* createdByAdminId
* status (draft/published)

#### 2) FormField (the questions inside the form)

Each template has many fields:

* fieldId
* label: “Is helmet worn?”
* type: checkbox / text / number / dropdown / photo
* required: true/false
* options: (for dropdown)
* order

So admin “creates a form” = we save:

* one row in **form_templates**
* many rows in **form_fields**

That’s the “one click create” you mean: it becomes stored instantly.

#### 3) FormSubmission (filled data)

When user fills the form, we save:

* submissionId
* templateId + version
* submittedByUserId
* plantId
* status: submitted/approved/rejected
* submittedAt

#### 4) SubmissionAnswer (actual answers)

Each answer saved as:

* submissionId
* fieldId
* value (stored as text/json)
* attachments (if any)

So every form can be different because **the structure is stored as rows**, not as new tables.
